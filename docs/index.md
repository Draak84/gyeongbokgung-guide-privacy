# Privacy Policy — Gyeongbokgung Guide

*Last updated: 2026-04-26*

This Privacy Policy describes how the mobile application **Gyeongbokgung Guide** ("the App") handles user information. The App is built to provide self-guided tour content for visitors to Gyeongbokgung Palace and **does not store user content data on any server**.

## 1. Information We Collect

| Type | When Collected | Purpose | User Consent |
|---|---|---|---|
| **Advertising Identifier (IDFA)** | On first launch, only if you grant permission via the iOS App Tracking Transparency dialog | Personalized ad serving via Google AdMob | Explicit consent. If declined, you continue to see non-personalized ads |
| **Approximate Location** | Only if you grant location permission | To show your current location on the palace map | Explicit consent. If declined, the map remains fully usable; only the "your location" indicator is hidden |
| **Diagnostic Data** (crash logs, device model, OS version) | Automatically when the App crashes | App stability via Firebase Crashlytics | Automatic. No personal identifiers collected |
| **Usage Analytics** (screen views, button taps) | Automatically during use | Feature improvement via Firebase Analytics | Automatic. No personal identifiers collected |

The App **does not directly collect or store** your name, email, phone number, contacts, photos, or call logs.

## 2. Your Rights

- **Decline App Tracking Transparency (ATT)**: iOS Settings → Privacy & Security → Tracking → toggle off for this App. Ads continue to be shown, but as non-personalized ads.
- **Decline location permission**: iOS Settings → Privacy & Security → Location Services → this App → Never. The map still works fully; only the "your location" indicator is hidden.
- **Push notifications**: This App does not use push notifications.

## 3. Third-Party Data Sharing

The App uses the following third-party services. Their respective privacy policies apply.

| Service | Provider | Purpose |
|---|---|---|
| **Google AdMob** | Google LLC | Ad serving (uses IDFA) — [Privacy Policy](https://policies.google.com/privacy) |
| **Google Maps SDK** | Google LLC | Map tile rendering — [Privacy Policy](https://policies.google.com/privacy) |
| **Firebase Crashlytics** | Google LLC | Crash reporting (no IDFA used) — [Privacy Policy](https://firebase.google.com/support/privacy) |
| **Firebase Analytics** | Google LLC | Usage pattern analysis (no IDFA used) — [Privacy Policy](https://firebase.google.com/support/privacy) |

These four services are the only paths through which any data leaves your device.

## 4. Data Retention

- **On-device**: Your selected language, visited landmarks, and ad/location permission states are stored locally in iOS UserDefaults. They are never transmitted externally.
- **On uninstall**: All on-device data is removed immediately when the App is deleted.
- **No backend server**: The App operates fully self-contained — palace content (text, images, maps) is bundled within the App. No content fetch over the network.

## 5. Children Under 13

The App is not intentionally targeted at children under 13. Advertisements served via AdMob may not be appropriate for children. Parental supervision is recommended.

## 6. Users in the EU/EEA (GDPR)

If you use the App from the EU or EEA, in line with the General Data Protection Regulation (GDPR), you may be presented with a separate consent dialog regarding personalized advertising. Declining personalized ads still allows normal App usage with non-personalized ads.

## 7. Policy Changes

Changes to this policy will be posted on this page and announced via App update. For material changes, in-App notice will also be provided.
